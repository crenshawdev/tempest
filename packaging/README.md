# Packaging & distribution (maintainer notes)

Tempest is **self-distributed**. After being delisted from the cosmic-utils Flatpak
pointer repo, releases ship through channels owned entirely here, all keyed off a
`vX.Y.Z` tag pushed to `github.com/crenshawdev/tempest`:

- **deb / rpm** — built by `.github/workflows/release.yml` with nfpm and attached to
  the GitHub Release. The `pkg.jcrenshaw.dev` aggregator re-signs and serves them.
- **Self-hosted Flatpak remote** — `packaging/flatpak/`, built and signed by the same
  workflow and published to GitHub Pages; the aggregator mirrors it.
- **AUR** — `packaging/aur/` (source package). See the note below: these files still
  reference the old GitLab source and need a migration pass.

Everything except the AUR is produced by one tag push, no manual step.

---

## deb / rpm / Flatpak (GitHub Actions)

A `v*` tag runs `release.yml`:

- **`package`** builds the release binary and packages an unsigned `.deb` and `.rpm`
  with nfpm (`packaging/nfpm.yaml`), attached to the tag's GitHub Release.
- **`flatpak`** builds the app into an archive-z2 OSTree repo with flatpak-builder
  (inside the flathub-infra privileged container), GPG-signs it, and publishes it to
  GitHub Pages under `/repo`.

The `pkg.jcrenshaw.dev` aggregator (repo `jcrenshaw-packages`) then assembles the
signed APT and RPM repos from the Release assets and `ostree pull --mirror`s the
Pages OSTree repo, serving all three at `pkg.jcrenshaw.dev/{deb,rpm,flatpak}`. Users
add one remote per format; see the top-level `README.md` for the user-facing commands.

**store key**:

```

```

In this repo it is the `GPG_SIGNING_KEY_B64` secret (base64 of the passphraseless
private key) and the `GPG_FINGERPRINT` variable. The aggregator imports the same key.
**Back up the private key somewhere safe** — losing it means every user must re-add
the remotes under a new key.

### When the version changes

1. Bump `version` in `Cargo.toml` (and refresh `Cargo.lock`).
2. Bump the manifest `tag` in `com.vintagetechie.CosmicExtAppletTempest.json` to the
   new `vX.Y.Z`. `packaging/flatpak/assert-manifest-tag.sh` fails the release if the
   manifest tag drifts from the release tag.
3. Regenerate offline crate sources only if `Cargo.lock` changed:
   ```bash
   python3 flatpak-cargo-generator.py Cargo.lock -o cargo-sources.json
   ```
   (`flatpak-cargo-generator.py` lives in flathub/flatpak-builder-tools; needs
   `aiohttp`, `PyYAML`, `tomlkit`.)
4. Add a `metainfo.xml` `<release>` entry and a `CHANGELOG.md` section.
5. Tag and push: `git tag -s -m "Tempest vX.Y.Z" vX.Y.Z && git push origin vX.Y.Z`.

### Local build check

To smoke-test the Flatpak build locally before tagging (what CI does, minus the
signing and Pages publish):

```bash
flatpak-builder --user --force-clean --repo=/tmp/tempest-repo \
    /tmp/tempest-build com.vintagetechie.CosmicExtAppletTempest.json
flatpak-builder --run /tmp/tempest-build \
    com.vintagetechie.CosmicExtAppletTempest.json cosmic-ext-applet-tempest
```

The user-facing `jcrenshaw.flatpakrepo` descriptor, carrying the store key, is
generated by the aggregator; it is not committed here.

## AUR (migration pending)

`packaging/aur/` holds the canonical `PKGBUILD` and `.SRCINFO` (the AUR repo is a
*separate* git repo this directory is copied into). These files still point at the
retired GitLab source and are pinned to an older `pkgver`. Migrating them means
repointing `url`/`source` to `github.com/crenshawdev/tempest`, bumping `pkgver`, and
regenerating checksums and metadata:

```bash
cd packaging/aur
# 1. repoint url= and source= to the GitHub archive, bump pkgver, reset pkgrel=1
# 2. refresh the source checksum:
updpkgsums                       # (pacman-contrib)
# 3. test a clean build:
makepkg -f
namcap PKGBUILD *.pkg.tar.zst    # optional lint
# 4. regenerate metadata:
makepkg --printsrcinfo > .SRCINFO
```

Publish to the AUR (needs your AUR account + SSH key registered):

```bash
git clone ssh://aur@aur.archlinux.org/cosmic-ext-applet-tempest.git aur-pkg
cp PKGBUILD .SRCINFO aur-pkg/
cd aur-pkg && git commit -am "Update to X.Y.Z" && git push
```

`depends` (`wayland`, `libxkbcommon`, `openssl`) were derived from the linked + dlopen'd
libraries of the built binary — re-check with `namcap`/`ldd` if dependencies change.
